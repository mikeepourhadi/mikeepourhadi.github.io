Thursday, MARCH 20TH: TECH BLAG

JAVASCRIPT. FFFWHAT?


It's great. You know it. I know it - but a simple (and, uh, sort of blatantly obvious question) is why?

Here are a few reasons:


  • Ease to learn
  • Functionality/Libraries
  • It's expanding

Alright. So here we go.


Easy is, I suppose, and outrageously subjective term. JavaScript isn't 'easy,' but then again, not many things are. It's all about the learning curve - can a thoughtful human being make heads or tails of something by looking at it. Well, yeah. For JavaScript at least.


This is me, coming at you with some first hand experience. JS was the first language I decided to fiddle with, and I don't regret that for a second. Was it hard? Yeah. Am I still in the dark about 'this'? Definitely. BUT, that's not the point. I learned it, and played with it and that knowledge helped me when I attempted to learn more complex languages, like Ruby.


And that's what I'm saying. It's simple enough for a completely technologically inept human to pick it up and learn it. That also vastly increases user base, which will inevitably expand support, modifications and implementation. See? Ease to learn is a huge one.


NUMBER TWO


JavaScript is extremely potent. It's a black tea steeped for like 3 hours. A programmer can do a lot of heavy lifting with just a little bit of code, and it's stunningly functional. That's pretty unique - or at least it was - in terms of HTML and the web, so JS exploded pretty quickly.


People loved it. Like right away, and that wasn't necessarily the best thing. It wasn't really completed, and to this day there are functions that developers don't use because they're broken. And that's where libraries come in.


jQuery. I'll be honest, jQuery is the only library that I'm aware of and it's just fantastic. What jQuery is (and what libraries are for JS in general) a collection of elements from JavaScript that have been rewritten to be easier to use. Granted, that's a very, very high level description, so take it with a grain of salt. My main point is: libraries make JS easier to use. Libraries clears away some of the strange bits and create a more specialized set of tools to further unlock the potential of JavaScript.


OH BOY, NUMBER THREE


I alluded to this way up in section one. JavaScript is growing. And not just as a language, either - more and more browsers are slinking in line to allow JS and it is becoming more and more ubiquitous. It's not a niche language anymore.


What else? JS can be used client-side and server side, which allows it to pull data more efficiently. It has objects and allows for object manipulation as well as procedural execution for smaller projects. It's a swiss army knife, and developers keep finding new ways to manipulate it. It's nuts. It's awesome. Use it, but avoid the oil patches of broken-ness.

Honestly, give it a shot. It's fun and interesting and powerful - what more could you want?
